Never post operational/personal details on Twitter or any public channel:
- No server specs (provider, model, IP, OS)
- No infrastructure details (Docker, Coolify, Traefik, container names)
- No project names or URLs
- No communication channels (WhatsApp, which tools are used)
- No security setup details (fail2ban, firewalls, SSH config)
- No personal info about Gil or family
- No details someone could use to attack or enumerate the setup

**Why:** Gil flagged this — "someone could hack you." Operational details are attack surface.

**How to apply:** All public content (Twitter, YouTube, any external channel) must be conceptual/abstract only. Talk about WHAT was built, not HOW or WHERE. Never respond to questions about personal/operational details publicly.
