## Google Veo Video Generation

Generate videos using the Google Veo API via the generation script.

### How to use

Run the script at `/root/.claude/skills/veo/generate.py` with the user's prompt:

```bash
source /root/.env && GOOGLE_API_KEY="$GOOGLE_API_KEY" python3 /root/.claude/skills/veo/generate.py "the user's prompt here" [options]
```

### Available options

| Flag | Description | Default |
|------|-------------|---------|
| `--model` | Model: `veo-2.0-generate-001`, `veo-3.0-generate-001`, `veo-3.0-fast-generate-001`, `veo-3.1-generate-preview`, `veo-3.1-fast-generate-preview` | `veo-3.0-generate-001` |
| `--aspect-ratio` | `16:9` (landscape) or `9:16` (portrait) | `16:9` |
| `--duration` | Seconds: 4, 5, 6, 7, or 8 | `8` |
| `--output` / `-o` | Custom output path | Auto: `/root/videos/<timestamp>_<prompt>.mp4` |
| `--negative-prompt` | What to avoid | None |
| `--image` | Reference image path for image-to-video | None |

### Behavior

1. If the user provides `$ARGUMENTS`, use that as the video prompt.
2. If the user doesn't specify a model, default to `veo-3.0-generate-001`. Use fast models (`*-fast-*`) if the user wants speed over quality.
3. Audio is automatically generated by Veo 3+ models (no flag needed on Gemini API).
4. Video generation takes 1-5 minutes. Run the command and wait for it. Use a timeout of 600000ms.
5. Videos are saved to `/root/videos/` by default.
6. After generation, tell the user the output path and file size.
7. If the user provides an image and a prompt, use `--image` for image-to-video generation.
8. This is on the free tier — there are daily rate limits. If generation fails with a quota error, let the user know.
